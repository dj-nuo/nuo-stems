### NUO-STEMS is "stuck" at "Checking License" or "License Check Failed"
There might be few potential causes:

1.  Your computer is not connected to the internet. NUO-STEMS requires internet connection on the startup to check the license.
2.  Your provider/system/WiFi-router blocks `https://nuo-stems.com` from being reachable.
    1.  Restart computer
    2.  **Restart WiFi router**
    3.  Launch NUO-STEMS
3.  Windows PowerShell is for some reason "broken" on your computer
    1.  Follow this guide to fix it: [link](https://support.slatedigital.com/hc/en-us/articles/20352080857363-Powershell-is-not-recognized)

If none of the above helps - try settings your DNS to Google DNS: 8.8.8.8 and 8.8.4.4. Instructions: [link](https://developers.google.com/speed/public-dns/docs/using)

###  NUO-STEMS crashes at launch
Try deleting these folders and then start NUO-STEMS again:

1. On Windows:
    1. `C:\Users\YOUR_USERNAME\AppData\Local\Programs\NUO-STEMS-4\` 
        - (or `NUO-STEMS-3` if you're using NUO-STEMS 3)
    2. And also remove: `C:\Users\YOUR_USERNAME\AppData\Roaming\NUO-STEMS-4\` 
        - (or `NUO-STEMS-3` if you're using NUO-STEMS 3)
2. On Mac OS:
    1. `/Users/YOUR_USERNAME/Library/Application Support/nuo-stems-4/` 
        - (or `nuo-stems-3` if you're using NUO-STEMS 3)

If it doesn't help:

1. Launch NUO-STEMS and let it "crash"
2. [Send me](https://www.facebook.com/djnuo) main.log file from:
    1. Windows logs location: `C:\Users\YOUR_USERNAME\AppData\Roaming\nuo-stems-4\logs\main.log` 
        - (or `nuo-stems-3` if you're using NUO-STEMS 3)
    2. Mac OS logs location: `/Users/YOUR_USERNAME/Library/Logs/nuo-stems-4/main.log` 
        - (or `nuo-stems-3` if you're using NUO-STEMS 3)

### My collection.nml got corrupted, how do I fix it?
NUO-STEMS 4 (and NUO-STEMS 3) automatically creates a backup of your collection.nml on every launch. If you have a problem with your collection.nml - try to revert it to the backup.

Backups are stored in the **"Collection Backups NUO-STEMS 4"** or **"Collection Backups NUO-STEMS 3"** folder in the same folder as your collection.nml. Example for Traktor 4.1.1:
- MacOS: `/Users/YOUR_USERNAME/Documents/Native Instruments/Traktor 4.1.1/Collection Backups NUO-STEMS 4` or `3`
- Windows: `C:\Users\YOUR_USERNAME\Documents\Native Instruments\Traktor 4.1.1\Collection Backups NUO-STEMS 4` or `3`
